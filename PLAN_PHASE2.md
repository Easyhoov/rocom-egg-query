# Phase 2 方案：精灵图鉴 + 孵蛋关联

## 数据源对比

| | floranix 官方数据 | 我们现有数据 |
|---|---|---|
| 精灵数量 | 468 只 | 1623 条繁殖记录 |
| 蛋组 | 15 组（官方） | 16 组（社区数据） |
| 可繁殖精灵 | 455 只 | 无直接标记 |
| 额外信息 | 属性、六维、进化链、形态、特性、图片、获取地点 | 仅孵蛋身高体重范围 |

**蛋组差异：** 官方有「大地组、魔力组、龙组」，我们有「乖乖组、巨龙组、怪兽组、矿石组、精灵组」。
**结论：** 以官方数据为主（图鉴展示），保留我们的繁殖数据（孵蛋计算核心）。

## 数据集成方案

### 数据文件
将 floranix 输出的 JSON 复制到项目 `data/` 目录：
- `data/spirits.json` — 468 只精灵全量数据（1.8MB）
- `data/official_egg_groups.json` — 官方 15 个蛋组
- 不复制 skills.json（19MB 太大，图鉴暂不需要技能详情）

### 图片
精灵图片使用腾讯官方 CDN（`static.gamecenter.qq.com`），不本地托管。
已有 rocomegg 本地图片（`static/creature-atlas/`）保留作 fallback。

## 后端 API 新增端点

### 1. 图鉴列表
```
GET /api/spirits?q=&attribute=&egg_group=&page=&page_size=
```
- `q` — 搜索（名称/编号）
- `attribute` — 属性筛选（光/草/火/水...18系）
- `egg_group` — 蛋组筛选
- `page` / `page_size` — 分页，默认 20 条

### 2. 精灵详情
```
GET /api/spirits/{spirit_id}
```
返回完整信息：属性、六维、特性、进化链、形态、蛋组、图片。

### 3. 官方蛋组列表
```
GET /api/official-egg-groups
```

### 4. 现有 /api/query 增强
孵蛋查询结果中附加精灵图片和编号（关联 spirits.json）。

## 前端页面

### 导航调整
首页从「孵蛋查询」单入口改为双入口卡片：
- 🔍 **孵蛋查询**（现有功能）
- 📖 **精灵图鉴**（新增）

### 图鉴页 `/compendium`
- **顶部**：搜索框 + 属性筛选（18 色标签）+ 蛋组筛选
- **主体**：精灵卡片网格（图片 + 名称 + 编号 + 属性标签 + 蛋组标签）
- **点击卡片** → 弹出/跳转精灵详情

### 精灵详情（弹窗或独立页）
- 精灵大图（官方 CDN）
- 基本信息：编号、属性、特性（含效果描述）、身高体重、获取地点
- 六维雷达图（HP/攻击/魔攻/防御/魔防/速度）
- 进化链（横向展示，带图片和等级）
- 蛋组标签
- **「查询孵蛋」按钮** → 跳转孵蛋查询页，预填身高体重

### 孵蛋查询页增强
- 查询结果的精灵卡片显示图片（关联 spirits.json）
- 结果可点击跳转到该精灵的图鉴详情

## 页面间关联流

```
首页 → 孵蛋查询 → 结果点击精灵 → 图鉴详情
首页 → 图鉴列表 → 精灵详情 → 「查询孵蛋」→ 孵蛋查询（预填数据）
首页 → 图鉴列表 → 蛋组筛选 → 同组精灵列表
```
