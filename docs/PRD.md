## Problem Statement

当前项目只有一个基础的孵蛋查询功能（输入身高体重 → 返回候选精灵列表），虽然有 R 值排序和蛋类型筛选等差异化优势，但在功能完整性和用户体验上与同类工具（rocomegg/洛克星盘）差距较大。用户需要一个功能全面、界面美观的孵蛋辅助工具站。

### 当前痛点

1. **查询结果不够直观** — 只有文字列表，没有匹配概率可视化
2. **命中精度不分明** — 不区分精准命中、容差命中、范围命中
3. **缺少蛋组/繁殖查询** — 用户查完蛋尺寸后，想知道怎么配对繁殖
4. **没有精灵图片** — 纯文字结果，辨识度低
5. **前端简陋** — 原生 HTML/JS，没有设计感

### 我们的优势（要保留）

1. **R 值计算排序** — R = 体重(kg)/身高(m)，按 r_diff 升序，直观判断匹配概率
2. **REST API 后端** — 可被飞书机器人、微信小程序等第三方调用
3. **数据量更大** — 1623 条记录 vs rocomegg 的 1416 条
4. **三态蛋类型筛选** — 神奇的蛋/炫彩蛋/不限，API 参数灵活

---

## Solution

将项目升级为功能完整的孵蛋工具站，在保留 API 后端优势的基础上，融合 rocomegg 的 UI 设计和功能亮点，分两期迭代完成。

### 一期（核心功能升级）

- 升级蛋尺寸查询：分层命中 + 概率可视化 + 精灵图片
- 新增蛋组查询 API
- 新增繁殖匹配查询 API
- 前端重写：Vue 3 + 现代 UI 设计
- 深色模式支持

### 二期（进阶功能）

- 异色孵化路线规划
- 精灵图鉴页面
- 分享长图功能
- 数据投稿入口

## User Stories

### 一期 User Stories

**蛋尺寸查询（升级）**

1. 作为玩家，输入蛋的身高和体重后，想看到"精准命中/容差命中/范围命中/近似候选"分层结果，以便快速判断哪些精灵概率最高
2. 作为玩家，想看到每个候选精灵的匹配概率进度条，以便直观比较
3. 作为玩家，想看到候选精灵的图片，以便确认是不是我想要的
4. 作为玩家，想继续使用 R 值排序功能（现有优势），以便更准确判断匹配度
5. 作为玩家，想按蛋类型（神奇的蛋/炫彩蛋/不限）筛选结果，以便缩小范围
6. 作为玩家，想用手机方便地查询，需要移动端适配的界面

**蛋组查询（新增）**

7. 作为玩家，输入精灵名字后，想看到它属于哪些蛋组，以便规划繁殖
8. 作为玩家，输入蛋组名字后，想看到该蛋组下的所有精灵，以便找配对对象
9. 作为玩家，想看到精灵的进化链信息，以便了解最终形态

**繁殖匹配查询（新增）**

10. 作为玩家，输入目标精灵（母系）后，想看到所有可配对的父系候选，以便选择繁殖对象
11. 作为玩家，想看到父系精灵的蛋组标签和进化链，以便确认配对可行性
12. 作为玩家，想看到"不可繁殖"的精灵提示（如迪莫），避免无效尝试

**API 后端（升级）**

13. 作为开发者，想通过 REST API 查询蛋组关系，以便集成到机器人
14. 作为开发者，想通过 REST API 查询繁殖匹配结果，以便自动化查询
15. 作为开发者，API 返回的 JSON 要保持向后兼容

**前端体验**

16. 作为用户，想切换深色/浅色/自动主题，以便在不同光线环境下舒适使用
17. 作为用户，想看到加载动画，避免查询时以为卡住了
18. 作为用户，想用 Tab/Enter 快捷键操作，提高效率

### 二期 User Stories

19. 作为玩家，想维护我的异色精灵清单，系统推荐下一步孵化目标
20. 作为玩家，想查看完整精灵图鉴，了解所有精灵的数据
21. 作为玩家，想把查询结果生成图片分享给朋友

## Implementation Decisions

### 架构决策

1. **前后端分离** — 保持 FastAPI 后端 + Vue 3 前端，通过 API 通信
2. **后端优先** — 先完善 API，再重写前端页面
3. **数据扩展** — 在 breeding.json 基础上，增加蛋组数据（可从 rocomegg 的 EGG_GROUP_INPUT 参考，但需自行验证数据准确性）

### 模块设计

#### 后端新增模块

| 模块 | 接口 | 说明 |
|------|------|------|
| 蛋组查询 | `GET /api/egg-group?name=魔力猫` | 按精灵名查蛋组 |
| 蛋组反查 | `GET /api/egg-group/members?group=动物组` | 按蛋组名查成员 |
| 繁殖匹配 | `GET /api/breeding?target=魔力猫` | 查父系候选列表 |
| 精灵详情 | `GET /api/pet/{name}` | 精灵详情（蛋组+进化链+尺寸） |

#### 后端升级模块

| 模块 | 变更 |
|------|------|
| `/api/query` | 增加分层命中逻辑：精准(完全匹配) → 容差1(±0.01m/±0.1kg) → 容差2(±0.02m/±0.2kg) → 范围命中 → 近似候选 |
| `/api/query` | 增加 `match_tier` 字段：`exact` / `tolerance1` / `tolerance2` / `range` / `nearest` |
| `/api/query` | 增加 `probability` 字段：基于匹配精度的概率估算 |

#### 前端重写

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 功能导航 + 数据统计 |
| 蛋尺寸查询 | `/size` | 分层命中 + 概率条 + 精灵图 |
| 蛋组查询 | `/group` | 按名字/蛋组双向查 |
| 繁殖匹配 | `/breeding` | 母系输入 → 父系候选 |

### 数据决策

1. **蛋组数据来源** — 需要新增 `egg_groups.json`，包含 16 个蛋组及其成员列表。参考来源：
   - rocomegg 的 `EGG_GROUP_INPUT`（需自行验证）
   - breeding.json 中的 `pet_egg_conf` 交叉验证
   - 社区 Wiki 数据补充

2. **进化链数据** — 需要新增 `evolution_chains.json`，记录精灵的进化关系

3. **不可繁殖列表** — 参考 rocomegg 的 `NO_BREED_PETS`（迪莫、里奥等 13 只）

### 技术栈决策

| 层 | 技术 | 理由 |
|----|------|------|
| 后端 | FastAPI (Python) | 现有，保持不变 |
| 前端 | Vue 3 + Vite + Element Plus | rocomegg 已验证可行，组件丰富 |
| 数据 | JSON 文件 | 简单够用，无需数据库 |
| 样式 | 自定义 CSS + Element Plus | 参考 Notion/Airtable 设计风格 |
| 部署 | 当前服务器直接运行 | Hermes 可直接操作 |

### API 兼容性

- 现有 `/api/query` 接口保持向后兼容，新增字段不影响旧调用方
- `GET /api/query?height=X&weight=Y` 返回格式不变，只增加字段
- 新接口使用 `/api/` 前缀，不与现有路由冲突

## Testing Decisions

### 后端测试

1. **查询逻辑测试** — 用已知身高体重验证返回结果正确性
   - 0.24m / 1.60kg → 应返回喵喵等
   - 分层命中逻辑验证
2. **蛋组查询测试** — 验证已知精灵的蛋组归属
3. **繁殖匹配测试** — 验证已知配对关系
4. **边界条件** — 身高体重为 0、负数、极大值
5. **API 兼容性** — 现有调用方格式不被破坏

### 前端测试

1. **响应式布局** — 手机(375px)、平板(768px)、桌面(1200px)
2. **交互流程** — 输入 → 查询 → 结果展示完整链路
3. **主题切换** — 浅色/深色/自动三档
4. **快捷键** — Enter 触发查询

## Out of Scope

### 一期不做

- ❌ 异色孵化路线规划（二期）
- ❌ 精灵图鉴/技能图鉴页面（二期）
- ❌ 分享长图功能（二期）

- ❌ 数据库迁移（继续用 JSON 文件）
- ❌ 用户认证/登录系统

### 明确不做的

- ❌ 不引入构建工具链的复杂度超出必要
- ❌ 不对接 B 站 WIKI API（有反爬机制）

- **推荐**: **Notion** 风格 — 温暖极简、衬线标题、柔和表面、适合数据密集工具
